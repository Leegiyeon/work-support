# Team Context Snapshot: UI polish and data-entry clarity

## Task statement
Improve work-support UI composition and alignment so buttons, widgets, dashboard panels, edit flows, and data-entry surfaces are balanced, clear, user-friendly, and suitable for entering real project/work/outcome/career data.

## Desired outcome
Existing screens feel like a practical project-management tool: dense but readable dashboard grids, clear primary/secondary/danger buttons, predictable edit/create flows, balanced panels, and tables/forms that help users input real data. No new product feature expansion; polish and usability only.

## Known facts/evidence
- Stack: Next.js frontend, FastAPI backend, PostgreSQL.
- Current UI files: `frontend/app/page.tsx`, `frontend/app/projects/page.tsx`, `frontend/app/projects/[projectId]/page.tsx`, `frontend/app/reports/page.tsx`, `frontend/app/globals.css`.
- User-project seed data exists and was verified visible through project list/detail/task/log/outcome/career screens.
- UI principle already requested: data-centered, navy, Jira-like, minimal long copy.
- Current working tree has prior uncommitted MVP/UI/seed changes; preserve them.

## Constraints
- Do not add unrelated new features.
- Do not remove existing functionality.
- Keep UI text concise and data-oriented.
- Prioritize real-data entry usability.
- Verify with frontend lint/typecheck and relevant backend seed test if touched.
